Inference Latency Fundamentals

Remember: Prefill is parallel, but decode is sequential.

Prefill

One big parallel pass over the prompt.

Decode

One token at a time. This is where latency lives.

prompt -> prefill -> decode loop (sample -> append -> repeat)
Timing: 40s. Click moments: callout, then prefill card, then decode card. Land on: decode dominates latency.

KV Cache Fundamentals

Without KV Cache

Recompute old context every token.

With KV Cache

Reuse old K/V. Compute only new token work.

Timing: 30-35s. Use contrast framing: without cache vs with cache. End with "cache is mandatory for long outputs".

KV Cache Cost Model

Memory grows roughly as:

KV bytes ~ batch_size x seq_len x num_layers x hidden_dim x 2(KV) x dtype_bytes
Tradeoff: Throughput and context rise together with active KV memory pressure.

Context up

Memory up.

Batch up

Throughput up, then pressure.

Precision down

Capacity back.

Timing: 45s. Point to formula once, then only interpret trend directions (up/up/down). Do not derive math live.

Speculative Decoding Overview

Main win: Accept multiple tokens per verifier pass when draft and target agree.

Draft model

Small and fast. Proposes K next tokens.

Verifier model

Large model checks and accepts valid prefix.

Speedup = more accepted tokens per verifier step
Timing: 40s. Explain draft/verifier in plain words. Emphasize accepted prefix is the key lever.

Verification Loop

  1. Draft proposes K tokens.
  2. Verifier checks the path.
  3. Accept until first mismatch.
  4. Verifier emits next token, repeat.

Higher agreement -> higher speedup.

Timing: 35-40s. Click each step. Keep rhythm tight: propose, verify, accept, continue.

Core Metrics

  • TTFT - first token latency
  • TPOT - per-token decode latency
  • Tokens/sec - throughput per GPU
  • Acceptance ratio - spec decode quality
  • Active KV bytes - memory pressure
  • p95 / p99 - user pain at tail
